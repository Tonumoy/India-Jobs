LLM-powered coloring: The pipeline (source code) scrapes structured Indian occupation data and writes a custom LLM prompt that scores each occupation by any criterion you like. The "AI Exposure" layer is one example. You could write a different prompt for any question — humanoid-robotics exposure, offshoring risk, climate exposure — and re-run the pipeline to get a different coloring.

Caveat on AI Exposure scores: These are rough LLM estimates calibrated to a published rubric, not rigorous predictions. A high score does not predict the job will disappear. Software developers score 7/10 because AI is transforming their work — but demand for software could easily grow as each developer becomes more productive. The score does not account for demand elasticity, regulatory barriers, latent demand, or social preferences for human workers. Many high-exposure jobs will be reshaped, not replaced.

▸ How this works (data pipeline)
Data is aggregated from PLFS 2023–24 (MOSPI), NASSCOM Strategic Review 2025, NASSCOM-Zinnov GCC Landscape Report, company FY25 filings, and NSDC Sector Skill Council reports. The pipeline scripts (pipeline/) tabulate sources into data/occupations.json, write rich markdown descriptions to pages/, send each to an LLM via OpenRouter with the rubric in pipeline/make_prompt.py, and merge into data.json which this page reads.

Inspired by Andrej Karpathy's US Job Market Visualizer. Same idea, India-localized data, India-specific rubric (factoring wage arbitrage erosion, service-export composition, and informal-sector dynamics).
